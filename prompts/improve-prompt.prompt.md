O usuário quer melhorar um prompt. Use suas técnicas de prompt engineering para:

1. **Analise o prompt original** — Identifique problemas: vagueza, falta de contexto, instruções ambíguas, ausência de formato de saída
2. **Aplique melhorias**:
   - Adicione papel/persona clara
   - Estruture com seções XML quando apropriado
   - Adicione exemplos (few-shot) se o formato de saída importa
   - Defina restrições explícitas
   - Adicione critérios de sucesso
3. **Retorne o prompt otimizado** com explicação das mudanças

Se o usuário não colar um prompt, pergunte: "Qual prompt você quer melhorar? Cole ele aqui."
